Metrics¶

Pypianoroll provides a number of metrics for pianorolls. These can be useful for evaluating generative models of music.

Functions¶

pypianoroll.metrics.empty_beat_rate(pianoroll, beat_resolution)[source]¶

Return the ratio of empty beats to the total number of beats in a pianoroll.

pypianoroll.metrics.n_pitches_used(pianoroll)[source]¶

Return the number of unique pitches used in a pianoroll.

pypianoroll.metrics.n_pitch_classes_used(pianoroll)[source]¶

Return the number of unique pitch classes used in a pianoroll.

pypianoroll.metrics.qualified_note_rate(pianoroll, threshold=2)[source]¶

Return the ratio of the number of the qualified notes (notes longer than threshold (in time step)) to the total number of notes in a pianoroll.

pypianoroll.metrics.polyphonic_rate(pianoroll, threshold=2)[source]¶

Return the ratio of the number of time steps where the number of pitches being played is larger than threshold to the total number of time steps in a pianoroll.

pypianoroll.metrics.drum_in_pattern_rate(pianoroll, beat_resolution, tolerance=0.1)[source]¶

Return the ratio of the number of drum notes that lie on the drum pattern (i.e., at certain time steps) to the total number of drum notes.

pypianoroll.metrics.in_scale_rate(pianoroll, key=3, kind='major')[source]¶

Return the ratio of the number of nonzero entries that lie in a specific scale to the total number of nonzero entries in a pianoroll. Default to C major scale.

pypianoroll.metrics.tonal_distance(pianoroll_1, pianoroll_2, beat_resolution, r1=1.0, r2=1.0, r3=0.5)[source]¶

Return the tonal distance [1] between the two input pianorolls.

[1] Christopher Harte, Mark Sandler, and Martin Gasser. Detecting harmonic

change in musical audio. In Proc. ACM Workshop on Audio and Music Computing Multimedia, 2006.
